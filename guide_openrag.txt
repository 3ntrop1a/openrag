Guide d'utilisation d'OpenRAG

OpenRAG est un système RAG (Retrieval-Augmented Generation) complet et open-source.

Fonctionnalités principales :
- Upload de documents aux formats PDF, DOCX, TXT et Markdown
- Extraction et découpage automatique du contenu en chunks
- Génération d'embeddings vectoriels avec sentence-transformers
- Stockage vectoriel dans Qdrant pour la recherche sémantique
- Génération de réponses avec Ollama, OpenAI ou Anthropic
- Architecture microservices scalable avec Docker

Architecture :
- API Gateway (FastAPI) : Point d'entrée principal sur le port 8000
- Orchestrator : Coordination du workflow RAG
- Embedding Service : Génération des embeddings vectoriels
- PostgreSQL : Métadonnées et suivi des documents
- Redis : Cache et file d'attente de tâches
- MinIO : Stockage objet S3-compatible pour les documents
- Qdrant : Base de données vectorielle pour la recherche sémantique
- Ollama : Serveur LLM local pour la génération de texte

Comment utiliser OpenRAG :
1. Uploadez vos documents via l'endpoint POST /documents/upload
2. Attendez que le traitement soit terminé (chunking + embeddings + indexation)
3. Posez vos questions via l'endpoint POST /query
4. Recevez des réponses contextualisées basées sur vos documents

L'interface Swagger est disponible sur http://localhost:8000/docs
La console MinIO est accessible sur http://localhost:9001
Le dashboard Qdrant est disponible sur http://localhost:6333/dashboard

Pour des performances optimales, ajustez les paramètres d'embedding et de chunking
selon la nature de vos documents. La taille des chunks par défaut est de 512 tokens
avec un chevauchement de 50 tokens.
